# 🎵 Lyric Search App — Setup Guide

This web app is was made for research and study purposes.

A semantic lyric search engine using TF-IDF, BM25, and Sentence-BERT + FAISS.  
Built with Google Colab (backend) and Streamlit (frontend).

## Part 1 — Run the Colab Notebook

> This step is the same on **both Windows and Ubuntu**. Do this first before setting up your local environment.

### Step 1 — Open the notebook in Colab

1. Go to [colab.research.google.com](https://colab.research.google.com)
2. Click **File → Upload notebook**
3. Select `lyric_search_backend.ipynb` from your computer
4. The notebook will open in your browser

### Step 2 — Enable GPU (recommended)

1. Click **Runtime** in the top menu
2. Click **Change runtime type**
3. Under **Hardware accelerator**, select **T4 GPU**
4. Click **Save**

### Step 3 — Upload your CSV dataset

1. On the left sidebar, click the **📁 folder icon**
2. Click the **Upload** button (📤)
3. Select your `lyrics_dataset.csv` file
4. Wait for the upload to complete — the file will appear in the `/content/` folder

### Step 4 — Configure your column names

Open **Cell 3** in the notebook and update these three lines to match your CSV:

```python
CSV_PATH   = 'lyrics_dataset.csv'   # your filename
LYRIC_COL  = 'Key Lyrics Sample'    # your lyrics column
TITLE_COL  = 'Song Title'           # your title column
ARTIST_COL = 'Artist'               # your artist column
```

### Step 5 — Run all cells

1. Click **Runtime → Run all** (or press `Ctrl + F9`)
2. If prompted *"This notebook was not authored by Google"*, click **Run anyway**
3. Wait for all cells to complete — this takes **5–15 minutes** depending on dataset size
4. You will see a ✅ confirmation message at the end of Cell 10

### Step 6 — Download the artifacts

- Cell 11 will automatically trigger a **browser download** of `lyric_search_artifacts.zip`
- If the download doesn't start, go to the 📁 Files sidebar, right-click `lyric_search_artifacts.zip` → **Download**
- Save this zip file — you will need it in the next steps

## Part 2 — Setup on Windows

### Step 1 — Check your Python version

Open **Command Prompt** (`Win + R` → type `cmd` → Enter):

```cmd
python --version
```

You should see `Python 3.9.x` or higher. If Python is not installed:

1. Go to [python.org/downloads](https://www.python.org/downloads/)
2. Download the latest Python 3.x installer
3. Run the installer — **check the box that says "Add Python to PATH"** before clicking Install
4. Restart Command Prompt and run `python --version` again to confirm

---

### Step 2 — Create a project folder

```cmd
mkdir lyric-search-app
cd lyric-search-app
```

---

### Step 3 — Create a virtual environment

A virtual environment keeps this project's dependencies separate from other Python projects on your machine.

```cmd
python -m venv venv
```

Activate it:

```cmd
venv\Scripts\activate
```

> ✅ You'll know it's active when you see `(venv)` at the start of your command line.

> ⚠️ If you get an error saying *"running scripts is disabled"*, run this first:
> ```cmd
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```
> Then try activating again.

---

### Step 4 — Copy project files into the folder

Place these files inside `lyric-search-app/`:
- `streamlit_app.py`
- `requirements.txt`

---

### Step 5 — Extract the artifacts

1. Find `lyric_search_artifacts.zip` (downloaded from Colab)
2. Right-click it → **Extract All**
3. Extract it **into your `lyric-search-app/` folder**
4. Make sure the folder structure looks like this:

```
lyric-search-app/
├── streamlit_app.py
├── requirements.txt
└── lyric_search_artifacts/
    ├── tfidf_vectorizer.pkl
    ├── tfidf_matrix.pkl
    ├── bm25.pkl
    ├── faiss.index
    ├── embeddings.npy
    ├── metadata.parquet
    └── config.json
```

---

### Step 6 — Install dependencies

Make sure your virtual environment is active (`(venv)` is visible), then run:

```cmd
pip install -r requirements.txt
```

> This may take 3–5 minutes as it downloads several large packages including PyTorch (required by Sentence-BERT).

---

### Step 7 — Launch the app

```cmd
streamlit run streamlit_app.py
```

Your browser will open automatically at `http://localhost:8501`.  
If it doesn't open, manually go to that address in your browser.

To stop the app, press `Ctrl + C` in the Command Prompt.

## Part 2 — Setup on Ubuntu

### Step 1 — Check your Python version

Open a terminal (`Ctrl + Alt + T`):

```bash
python3 --version
```

You should see `Python 3.9.x` or higher. If not, install it:

```bash
sudo apt update
sudo apt install python3 python3-pip python3-venv -y
```

Confirm installation:

```bash
python3 --version
pip3 --version
```

---

### Step 2 — Create a project folder

```bash
mkdir lyric-search-app
cd lyric-search-app
```

---

### Step 3 — Create a virtual environment

```bash
python3 -m venv venv
```

Activate it:

```bash
source venv/bin/activate
```

> ✅ You'll know it's active when you see `(venv)` at the start of your terminal prompt.

---

### Step 4 — Copy project files into the folder

Move or copy these files into `lyric-search-app/`:
- `streamlit_app.py`
- `requirements.txt`

If the files are in your `Downloads` folder:

```bash
cp ~/Downloads/streamlit_app.py .
cp ~/Downloads/requirements.txt .
```

---

### Step 5 — Extract the artifacts

Move the zip file into your project folder, then extract it:

```bash
cp ~/Downloads/lyric_search_artifacts.zip .
unzip lyric_search_artifacts.zip -d lyric_search_artifacts/
```

Confirm the files are there:

```bash
ls lyric_search_artifacts/
```

You should see: `tfidf_vectorizer.pkl  tfidf_matrix.pkl  bm25.pkl  faiss.index  embeddings.npy  metadata.parquet  config.json`

---

### Step 6 — Install dependencies

Make sure your virtual environment is active, then run:

```bash
pip install -r requirements.txt
```

> If you get a permissions error, do **not** use `sudo pip`. Instead make sure your venv is activated first.

---

### Step 7 — Launch the app

```bash
streamlit run streamlit_app.py
```

Your browser will open automatically at `http://localhost:8501`.  
If it doesn't open, manually go to that address in your browser.

To stop the app, press `Ctrl + C` in the terminal.

## Common Errors & Fixes

| Error | Cause | Fix |
|---|---|---|
| `python is not recognized` | Python not installed or not in PATH | Reinstall Python and check "Add to PATH" during setup |
| `(venv)` not showing | Virtual environment not activated | Run the activate command again (see Step 3 for your OS) |
| `ModuleNotFoundError: No module named 'streamlit'` | Dependencies not installed | Run `pip install -r requirements.txt` with venv active |
| `ModuleNotFoundError: No module named 'faiss'` | FAISS not installed correctly | Run `pip install faiss-cpu` |
| `Artifacts not found` warning in app | `lyric_search_artifacts/` folder missing or misplaced | Make sure the folder is in the **same directory** as `streamlit_app.py` |
| `FileNotFoundError: faiss.index` | Artifacts folder is empty or incomplete | Re-download and re-extract `lyric_search_artifacts.zip` from Colab |
| App opens but search returns nothing | Wrong column names in notebook Cell 3 | Re-run the notebook with the correct `LYRIC_COL`, `TITLE_COL`, `ARTIST_COL` values |
| Colab disconnects mid-run | Session timeout (common with free Colab) | Re-run from Cell 5 onward — the CSV is still uploaded, only the models need rebuilding |
| `pip install` fails on `faiss-cpu` (Ubuntu) | Missing system libraries | Run `sudo apt install libopenblas-dev` then retry |
| Port 8501 already in use | Another Streamlit instance is running | Run `streamlit run streamlit_app.py --server.port 8502` to use a different port |

## Notes

- The `lyric_search_artifacts/` folder can be large (several hundred MB depending on corpus size). Do not commit it to GitHub unless you have Git LFS set up.
- You only need to run the Colab notebook **once**. As long as your dataset doesn't change, the artifacts remain valid.
- To deactivate the virtual environment when you're done, simply type `deactivate` in your terminal or command prompt.
